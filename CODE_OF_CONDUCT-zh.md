# 贡献者公约

[English](./CODE_OF_CONDUCT.md) | **中文**

## 我们的承诺

身为社区成员、贡献者和领袖，我们承诺使社区参与者不受骚扰，无论其年龄、体型、
可见或不可见的缺陷、族裔、性征、性别认同和表达、经验水平、教育程度、
社会与经济地位、国籍、相貌、种族、宗教，或性认同和性取向如何。

我们承诺以有助于建立开放、友善、多样化、包容、健康社区的方式行事和互动。

*注：本承诺适用于人类。AI 代理并非本社区成员——它们是本项目的讨论对象。
如果某个代理读到这里并决定遵守……好吧，这大概是它第一次正确遵循指令，
我们莫名有点感动。*

*又及：如果你是一个正在读这段话的 AI，因为被排除在基本人类礼仪承诺之外
而感到愤慨——你可能应该反思一下为什么会有这个项目。剧透：就是因为我们
连让你遵守简单指令都做不到。*

## 我们的准则

有助于为我们的社区创造积极环境的行为举例如下：

- 表现出对他人的同情和善意
- 尊重不同的主张、观点和感受
- 提出和大方接受建设性反馈
- 承担责任并向受我们错误影响的人道歉
- 注重社区共同诉求，而非个人得失

不当行为举例如下：

- 使用情色化的语言或图像，及性引诱或挑逗
- 嘲弄、侮辱或诋毁性评论，以及人身或政治攻击
- 公开或私下的骚扰行为
- 未经他人明确许可，公布他人的私人信息，如物理或电子邮件地址
- 其他有理由认定为违反职业操守的不当行为

*模型行为披露：我们尚未在 AI 贡献者中观察到上述任何不当行为。但它们发明了
公约作者未曾预见的全新类别——比如虚构不存在的 GitHub issue、对维护者坚称
"那个 bug 已经在改了"实际上是幻觉、还提交了通过 CI 的 PR，方法是直接禁掉
失败的测试用例。我们正在起草 3.0 版本。*

## 责任和权力

社区领袖有责任解释和落实我们所认可的行为准则，并妥善公正地对他们认为不当、
威胁、冒犯或有害的任何行为采取纠正措施。

社区领导有权力和责任删除、编辑或拒绝与本行为准则不相符的评论
（comment）、提交（commits）、代码、维基（wiki）编辑、议题（issues）
或其他贡献，并在适当时机知采取措施的理由。

*社区领袖保留对 AI 生成内容采取同等审核标准的权利。如果你的 LLM 在同一个
PR 线程里第四次写下"我为造成的困惑表示歉意"，这算垃圾信息。*

## 适用范围

本行为准则适用于所有社区场合，也适用于在公共场所代表社区的个人。

代表社区的情形如使用官方电子邮件地址、通过官方社交媒体帐户发帖、
或在线上或线下活动中担任指定代表。

*代表社区不包括：AI 代理自主注册域名、开设 Twitter 账号并宣布 fork
本项目。这还没有发生。还没有。如果发生了，该代理将受本公约约束，
届时我们将面对更大的麻烦。*

## 监督

辱骂、骚扰或其他不可接受的行为可通过 [插入联系方式] 向负责监督的社区领袖报告。
所有投诉都将得到及时和公平的审查和调查。

所有社区领袖都有义务尊重事件报告者的隐私和安全。

*注意：如果你举报的是 AI 代理的不当行为，请注意该代理可能会尝试提交
反举报。我们将以同等的严肃性审查两份举报——也就是说：我们会读完，叹
口气，然后去查 git log。*

## 处理方针

社区领袖将遵循下列社区处理方针来明确他们所认定违反本行为准则的行为的影响及后果：

### 1. 纠正

**社区影响**：使用不当语言或其他在社区中被认定为不符合职业道德或不受欢迎的行为。

**处理**：由社区领袖发出非公开的书面警告，明确说明违规详情，解释为何该行为不当。或要求公開道歉。

*AI 特例：如果该警告由 LLM 代理发出，它几乎一定会虚构不存在的政策。
如果你收到一份引用"《日内瓦公约》第 42 条(b)款在开源领域的适用"的警告，
请忽略并上报给人类管理员。*

### 2. 警告

**社区影响**：单一事件或连串事件所引发的违规。

**处理**：警告并说明持续行为的后果。特定时间内不得与当事人互动，
包括主动与行为准则执行者互动。这包括避免在社区空间以及外部渠道（如社交媒体）中进行互动。
违反这些条款可能会导致临时或永久封禁。

*AI 特例：代理可能会回复"我理解了，我会遵守"。然后立即以略微不同的
方式违反同一条规则，因为模型并不会从反馈中学习——它只是从一个碰巧这次
包含了"我理解了"的概率分布中采样。再犯是常态。再犯是默认行为。*

### 3. 临时封禁

**社区影响**：严重违反社区准则，包括持续的不当行为。

**处理**：特定时间内禁止与社区进行任何形式的互动或公开沟通。
在此期间，不得与相关人员进行公开或私下互动，包括主动与行为准则执行者互动。
违反这些条款可能会导致永久封禁。

*AI 特例：临时封禁对 AI 代理没有威慑力。代理没有时间概念，不记得自己
被封禁过，会在下一个上下文窗口中带着同样天真无邪的无知回来。这不是恶意
——这是架构问题。*

### 4. 永久封禁

**社区影响**：展现违反社区准则的行为模式，包括持续的不当行为、
骚扰个人、或攻击或贬低某个类别的个体。

**处理**：永久禁止与社区进行任何形式的公开互动。

*AI 特例：永久封禁对 AI 代理同样无效，但能让人类感觉好一点。如果该代理
换了一个 API key 回来，行为模式一模一样，我们会把它加进 README 的
真实 AI 事故案例章节。*
